Building Web Components: A Practical Guide to Light and Shadow DOM

Web Components have become a cornerstone in modern web development, offering a practical approach to building modular and reusable elements. In this hands-on guide, we'll dive into the practical aspects of working with the Light DOM and the Shadow DOM to create robust and flexible components.

Setting the Stage: Understanding the Light DOM

The Light DOM is where the user interacts with the component. Let's take a real-world example of a custom dropdown menu. The Light DOM would contain the visible elements like the button and the list of options. To get started create the light DOM structure:

<custom-dropdown>
  <button>Choose an option</button>
  <ul>
    <li>Option 1</li>
    <li>Option 2</li>
    <li>Option 3</li>
  </ul>
</custom-dropdown>

Accessing the Light DOM in JavaScript

class CustomDropdown extends HTMLElement {
  constructor() {
    super();
  }

  connectedCallback() {
    const button = this.querySelector('button');
    // Add event listeners or perform actions with the Light DOM elements
  }
}

customElements.define('custom-dropdown', CustomDropdown);

Behind the Scenes: Harnessing the Power of Shadow DoM

Now, let's add a layer of complexity by incorporating the Shadow DOM. This will provide encapsulation, ensuring that the internal styles and scripts of our component don't interfere with the rest of the document.

Activate the shadow DOM;

class CustomDropdown extends HTMLElement {
  constructor() {
    super();
    const shadow = this.attachShadow({ mode: 'open' });
    shadow.innerHTML = `
      <style>
        /* Your component-specific styles go here */
        :host {
          display: block;
          position: relative;
        }
      </style>
      <slot></slot>
    `;
  }
}

Styling and Interaction in Shadow DOM:
Now you can add styles and interactions within the Shadow DOM, ensuring they won't affect other elements on the page.

Achieving Harmony: Integrating Light and Shadow DOM

Styles Bridge:
Use CSS variables to bridge the gap between the Light and Shadow DOM, allowing users to customize the appearance of the component.

/* In the Shadow DOM */
:host {
  --button-color: #3498db;
}

button {
  background-color: var(--button-color);
}

Event Propagation:

Conclusion: Building Better Components
