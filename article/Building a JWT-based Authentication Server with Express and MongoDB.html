Building a JWT-based Authentication Server with Express and MongoDB

In this article, we'll explore the creation of a simple yet robust authentication server using JavaScript, Express, and MongoDB. This server utilizes JSON Web Tokens (JWT) for secure user authentication. Let's break down the key components of the project and understand the workflow.

Project Structure

The project structure is well-organized:

> auth-server: Main project folder.
> authV1.js: Express router for authentication routes.
> constants.js: Defines symbolic constants for error and success scenarios.
> helpers: Folder containing utility functions for authentication (auth.js) and database
> operations (db.js).
> index.js: Main entry point for the server.
> userSchema.js: MongoDB user schema.

Other files for configuration and package management.

Authentication Helper Functions (auth.js)

The auth.js file in the helpers folder serves as a utility module, providing essential functions for handling authentication within our server. From JWT token generation to regex validation of usernames and passwords, these functions play a crucial role in ensuring the security and integrity of the authentication process. Let's dive into the key functions within this file.

Token Generation and Verification:

// helpers/auth.js
import dotenv from "dotenv";
import jwt from "jsonwebtoken";
import {
  INVALID_TOKEN_ERROR,
  TOKEN_DURATION,
  TOKEN_EXPIRED_ERROR,
} from "../constants.js";

if (!process.env.PRODUCTION) {
  dotenv.config();
}

const helper = {
  middleware: {
    // ... middleware functions
  },
  token: {
    /**
     * @param {string} username
     * @returns {string | Symbol("InvalidUsernameError")}
     * @description Generate a JWT token with the username
     */
    generateJWT(username) {
      if (!username || !this.regex.verifyUsernameString(username))
        return INVALID_USERNAME_ERROR;

      return jwt.sign({ username }, process.env.JWT_SECRET, {
        expiresIn: TOKEN_DURATION,
      });
    },

    /**
     * @param {string} token - jwt: provided as authorization header by the client
     * @returns { jwt | Symbol("InvalidTokenError") | Symbol("TokenExpiredError") }
     *
     * @typedef {Object} decoded
     * @property {string} username
     */
    verifyJWT(token) {
      if (!token) return INVALID_TOKEN_ERROR;
      return jwt.verify(token, process.env.JWT_SECRET, (err, decoded) => {
        if (decoded) return decoded;
        else if (err?.name === "TokenExpiredError") return TOKEN_EXPIRED_ERROR;
        return INVALID_TOKEN_ERROR;
      });
    },
  },
  // ... regex validation functions
};

export default helper;

This section of the file defines the functions responsible for generating and verifying JWT tokens. The generateJWT function takes a username, ensures its validity, and returns a signed JWT token. The verifyJWT function, given a token, verifies its authenticity and expiration status, returning the decoded information or specific error symbols.


Regex Validation:

// ... (inside the helper object)
regex: {
  /**
   * @param {string} username
   * @returns {boolean}
   * @description Checks if the username is valid
   * @validUsername - starts with a letter, contains only letters, numbers and underscores, length 1-16
   */
  verifyUsernameString(username) {
    if (!username) return false;
    const regex = /^[a-zA-Z][a-zA-Z0-9_]{0,16}$/;
    return regex.test(username);
  },

  /**
   * @param {string} password
   * @returns {boolean}
   * @description Checks if the password is valid
   * @validPassword - contains only letters, numbers, underscores, @ and #, length 8-16
   */
  verifyPasswordString(password) {
    if (!password) return false;
    const regex = /^[a-zA-Z0-9_@#]{8,16}$/;
    return regex.test(password);
  },
},

These regex validation functions ensure that both usernames and passwords adhere to specified criteria. The verifyUsernameString function checks if a username starts with a letter, contains only letters, numbers, and underscores, and has a length between 1 and 16 characters. The verifyPasswordString function validates that a password contains only specified characters and has a length between 8 and 16 characters.

Database Operations Helper (db.js)

The db.js file, residing in the helpers folder, serves as the bridge between our authentication server and the MongoDB database. It contains essential functions for performing various database operations, including adding users, updating information, deleting users, and retrieving user data. Let's explore the key functionalities within this vital helper file.


Adding a new user to DB:

// helpers/db.js
import bcrypt from "bcrypt";
import dotenv from "dotenv";
import {
  DB_WRITE_FAILED_ERROR,
  INVALID_PASSWORD_ERROR,
  INVALID_USERNAME_ERROR,
  SUCCESS,
  USER_ALREADY_EXISTS_ERROR,
} from "../constants.js";

import User from "../userSchema.js";
import helper from "./auth.js";

if (!process.env.PRODUCTION) {
  dotenv.config();
}

const HASH_SALT = 10;

const db = {
  /**
   * @param {string} username
   * @param {password} password
   * @returns {Symbol("Success") | Symbol("InvalidUsernameError") | Symbol("InvalidPasswordError") | Symbol("UserAlreadyExistsError") | Symbol("DBWriteFailedError")}
   * @description Add a new user to the database
   */
  async addUser(username, password) {
    if (!helper.regex.verifyUsernameString(username))
      return INVALID_USERNAME_ERROR;
    if (!helper.regex.verifyPasswordString(password))
      return INVALID_PASSWORD_ERROR;
    if (await this.getUser(username)) return USER_ALREADY_EXISTS_ERROR;

    const passwordHash = await bcrypt.hash(password, HASH_SALT);
    const user = new User({
      username,
      password: passwordHash,
    });

    return user
      .save()
      .then(() => SUCCESS)
      .catch(() => DB_WRITE_FAILED_ERROR);
  },
  // ... other database operations
};

export default db;

This function, addUser, handles the addition of a new user to the database. It first validates the username and password using the regex functions from the auth.js helper. If the username is already taken, it returns USER_ALREADY_EXISTS_ERROR. If the user does not exist, it creates a hashed password using bcrypt, creates a new User instance, and attempts to save it to the database. The result determines whether the operation was successful (SUCCESS) or if it encountered an error (DB_WRITE_FAILED_ERROR).


Updating User Info:

/**
 * @param {string} username
 * @param {string | undefined} password - optional
 * @returns {Symbol("Success") | Symbol("InvalidUsernameError") | Symbol("InvalidPasswordError") | Symbol("DBWriteFailedError")}
 * @description Update a user's username and/or password
 */
async function updateUser(username, password) {
  if (!helper.regex.verifyUsernameString(username))
    return INVALID_USERNAME_ERROR;
  if (!password) return await User.updateOne({ username }, { username });

  if (password && !helper.regex.verifyPasswordString(password))
    return INVALID_PASSWORD_ERROR;

  const passwordHash = await bcrypt.hash(password, HASH_SALT);
  return User.updateOne({ username }, { username, password: passwordHash })
    .then(() => SUCCESS)
    .catch(() => DB_WRITE_FAILED_ERROR);
}

This updateUser function allows for the modification of a user's username and/or password. It performs similar validation as the addUser function, ensuring the new username and password (if provided) meet the required criteria. The user information is then updated in the database, with the result determining the success or failure of the operation.


Deleting user from DB:

/**
 * @param {string} username
 * @returns {Symbol("Success") | Symbol("InvalidUsernameError") | Symbol("DBWriteFailedError")}
 * @description Delete a user from the database
 */
async function deleteUser(username) {
  return User.deleteOne({ username })
    .then(() => SUCCESS)
    .catch(() => DB_WRITE_FAILED_ERROR);
}

The deleteUser function removes a user from the database based on their username. It returns SUCCESS if the deletion is successful, INVALID_USERNAME_ERROR if the username is invalid, and DB_WRITE_FAILED_ERROR if there's an issue with the database write operation. 


Retrieving User Information:

/**
 * @param {string} username
 * @returns {object | Symbol("InvalidUsernameError")} - user object or INVALID_USERNAME_ERROR symbol
 * @description Get a user from the database
 */
async function getUser(username) {
  const user = await User.findOne({ username });
  if (user) return user;
  return INVALID_USERNAME_ERROR;
}

The getUser function retrieves a user from the database based on their username. It returns the user object if found or INVALID_USERNAME_ERROR if the username is not valid. 


Comparing Passwords:

/**
 * @param {string} password - plain text password from the client
 * @param {string} hash - hashed password
 * @returns {Promise<boolean>} - true if the password matches the hash, false otherwise
 * @description Compare a password with a hash
 */
async function comparePassword(password, hash) {
  return await bcrypt.compare(password, hash);
}

The comparePassword function uses bcrypt to compare a plain text password with its hashed counterpart. It returns true if the password matches the hash and false otherwise.

Authentication Process Overview

To illustrate the authentication process within our server, let's use a symbolic diagram that outlines the key steps and interactions. This visual representation will provide a high-level overview of how the components work together to ensure secure user authentication.

Client Request (Login/Register):

The process begins when a client (e.g., a web browser or mobile app) sends a request to the server for authentication. This request can be either a login or registration attempt.

Express Router (authV1.js):
The request is directed to the authentication routes defined in the authV1.js file using Express Router. This file contains routes for login, registration, and middleware for token validation.

Middleware for Token Validation: 
If the request involves a secured route, the middleware checks for a valid JWT token in the request header. If the token is invalid or expired, the middleware allows access to the route; otherwise, it returns a response indicating that the user is already logged in.

Login Route: 
In the case of a login request, the server validates the provided username and password. If successful, a JWT token is generated using the auth.js helper functions, and the token along with user details is sent back to the client.

Registration Route:
For a registration request, the server validates the username and password. If the credentials are valid, the server adds a new user to the database using the db.js helper functions. If successful, a JWT token is generated, and user details are sent back to the client.

User Data Interaction (db.js): 
The db.js file serves as a helper for interacting with the MongoDB database. It contains functions for adding, updating, deleting, and retrieving user data. These operations ensure the integrity and security of user-related actions.

JWT Token Generation and Verification (auth.js):
The auth.js file provides essential functions for generating and verifying JWT tokens. These functions ensure secure communication between the server and the client, allowing for stateless and secure user authentication.

User Schema and Constants: 
The userSchema.js file defines the structure of user data in the MongoDB database. It ensures consistency in the storage of user information. Meanwhile, the constants.js file holds symbolic constants representing various states and errors within the authentication process.

Database Interaction (MongoDB): 
User data is stored and retrieved from the MongoDB database based on the defined schema. This ensures that user information remains structured and organized.

Response to Client: 
The server sends an appropriate response back to the client based on the success or failure of the authentication process. This response includes relevant information such as user details and JWT tokens.
