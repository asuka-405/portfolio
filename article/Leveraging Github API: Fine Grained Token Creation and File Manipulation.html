Leveraging Github API: Fine Grained Token Creation and File Manipulation

GitHub API is a powerful tool that allows developers to interact with repositories programmatically. In this article, we'll explore two essential aspects: creating fine-grained tokens for specific use cases and performing file operations using PUT and GET requests.

Creating Fine-Grained Tokens

GitHub tokens provide authentication and authorization. For fine-grained control, generate a token with only necessary scopes. Navigate to Settings > Developer settings > Personal access tokens on GitHub. Create a new token with minimal permissions for security.

Uploading and Editing Files with GitHub API

To upload or edit a file, use the PUT request. Below is a sample JavaScript code using XMLHttpRequest: 

async function uploadToGithub(uname, token, repo, filePath, content, sha) {
    const url = `https://api.github.com/repos/${uname}/${repo}/contents/${filePath}`

    // Encode content to base64
    content = btoa(content)

    const xhr = new XMLHttpRequest()
    xhr.open("PUT", url, true)
    xhr.setRequestHeader("Authorization", `token ${token}`)
    xhr.setRequestHeader("Content-Type", "application/json")
    xhr.setRequestHeader("Accept", "application/vnd.github.v3+json")

    xhr.onload = function () {
        if (xhr.status >= 200 && xhr.status < 300) {
            alert(`File uploaded successfully`)
            console.log(JSON.parse(xhr.responseText))
        } else {
            alert(`Error ${xhr.status}: ${xhr.responseText}`)
            console.log(JSON.parse(xhr.response))
        }
    }

    const data = {
        message: "File uploaded or edited",
        content,
        branch: "main",
    }

    if (sha) data.sha = sha
    xhr.send(JSON.stringify(data))
}
